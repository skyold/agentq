# Hyper Alpha Arena 项目概述

## 项目简介

**Hyper Alpha Arena** 是一个生产级的 AI 驱动加密货币永续合约交易平台，支持 **Hyperliquid** 和 **Binance Futures** 两大交易所。

### 核心理念

让大型语言模型（LLMs）自主执行加密货币交易策略。平台支持 GPT-5、Claude、Deepseek 等 AI 模型，基于实时市场数据做出智能交易决策并自动执行。

**官网**: https://www.akooi.com/

## 核心特性

### 1. 两种交易模式

| 模式 | 适用场景 | 工作原理 |
|------|----------|----------|
| **AI Trader** | 需要市场理解力的策略（新闻、情绪、复杂判断） | 用自然语言描述策略，AI 实时分析并决策 |
| **Program Trader** | 固定规则策略（技术指标、价格水平） | 用 Python 代码定义规则，支持历史回测，毫秒级执行 |

### 2. 因子智能系统

- **86 个内置因子**：涵盖动量、趋势、波动率、成交量、微观结构等类别
- **机构级评估指标**：IC、ICIR、胜率、衰减半衰期
- **自定义表达式引擎**：69 个函数支持创建自定义因子
- **AI 因子挖掘**：Hyper AI 可搜索网络量化研究论文，提取因子想法并用实时数据验证

### 3. 市场流信号监控

- 自动监控机构订单流、未平仓量激增、资金费率极端值
- 市场结构变化时触发交易
- 任何因子都可作为触发条件，提供精确的百分位分布（P5-P95）

### 4. AI 助手系统

- **Hyper AI**：中央协调器，具备工具使用能力
- **Signal AI**：通过对话设计市场流信号条件
- **Prompt AI**：实时预览变量，优化 AI Trader 策略提示词
- **Program AI**：编写、调试、回测 Python 交易策略
- **Attribution AI**：诊断策略表现，识别弱点并优化

### 5. 技能系统（Skill System）

内置模块化工作流指南，通过 `/command` 命令或自然语言描述自动加载：

| 命令 | 技能 | 功能 |
|------|------|------|
| `/prompt` | Prompt Strategy Setup | 从零创建 AI 决策交易策略 |
| `/program` | Program Strategy Setup | 构建 Python 交易程序 |
| `/market` | Market Analysis | 多源数据综合市场分析 |
| `/review` | Performance Review | 分析交易结果并优化 |
| `/diagnose` | Trader Diagnosis | 系统性诊断交易员为何未触发 |
| `/factor` | Factor Mining | AI 引导因子发现流程 |
| `/memory` | Memory Management | 管理 AI 对用户的记忆 |

### 6. 多交易所支持

- **Hyperliquid Testnet**：免费模拟交易，真实市场机制，无需 KYC
- **Hyperliquid Mainnet**：去中心化永续合约交易所，1-50 倍杠杆，钱包认证
- **Binance Futures**：全球最大加密衍生品交易所，USDT-M 永续合约，API Key 认证

### 7. 多模型 LLM 支持

支持任何 OpenAI API 兼容的语言模型：
- **Deepseek**（推荐）：交易场景性价比高
- **OpenAI**：GPT-5 系列、o1 系列、GPT-4o
- **Anthropic**：Claude 系列
- **自定义 API**：任何 OpenAI 兼容端点

## 技术架构

### 后端技术栈

- **框架**: FastAPI (Python)
- **数据库**: PostgreSQL 14
- **ORM**: SQLAlchemy 2.0
- **实时通信**: WebSocket
- **任务调度**: APScheduler
- **AI 集成**: 多 LLM 提供商抽象层
- **加密**: 敏感数据加密存储（API Key、私钥）

### 前端技术栈

- **框架**: React 18 + TypeScript
- **构建工具**: Vite 4
- **UI 库**: shadcn/ui + Tailwind CSS
- **状态管理**: React Context API
- **图表**: Chart.js, Lightweight Charts, Recharts
- **国际化**: i18next
- **代码编辑**: Monaco Editor

### 基础设施

- **容器化**: Docker + Docker Compose
- **包管理**: pnpm (前端), uv/pip (后端)
- **热重载**: 开发模式支持前后端热更新

## 核心功能模块

### 1. AI 决策引擎

- 多 LLM 提供商支持（OpenAI、Claude、Deepseek 等）
- 流式响应生成
- 上下文压缩（长对话管理）
- 工具调用系统（市场数据查询、因子分析等）
- 记忆系统（用户偏好、交易历史）

### 2. 交易执行系统

- 订单管理（创建、修改、取消）
- 持仓跟踪
- 多账户隔离
- 风险控制（杠杆限制、保证金监控）
- 自动交易触发（时间间隔、信号触发）

### 3. 市场数据系统

- K 线数据采集（WebSocket 实时 + REST 历史）
- 市场流指标（CVD、订单不平衡、深度比率）
- 技术指标计算（基于 pandas_ta，Numba JIT 加速）
- 市场状态识别（震荡、趋势）

### 4. 因子系统

- 因子注册表（86+ 内置因子）
- 因子有效性评估（IC、ICIR 分析）
- 自定义因子表达式引擎
- 因子数据存储和查询

### 5. 信号系统

- 信号定义（基于市场流指标）
- 信号池管理（OR/AND 逻辑组合）
- 边缘触发检测（避免重复触发）
- 触发日志和归因分析

### 6. 回测系统

- 历史数据回放
- 虚拟账户模拟
- 执行模拟器（滑点、手续费）
- 绩效评估指标

### 7. 分析系统

- 归因分析（按交易对、触发类型、时间段、因子）
- 资产曲线计算
- 交易回放
- Prompt 回测

### 8. Bot 集成

- Telegram Bot（命令交互、推送通知）
- Discord Bot（Gateway 连接）
- 事件推送系统

## 数据流示例

### AI 交易决策流程

```
1. 策略触发（时间间隔 or 信号池触发）
    ↓
2. 收集市场数据（K 线、持仓、新闻）
    ↓
3. 构建 Prompt 上下文
    ↓
4. 调用 LLM API（流式响应）
    ↓
5. 解析 AI 决策（买/卖/观望）
    ↓
6. 执行订单（通过交易所适配器）
    ↓
7. 记录决策日志（用于归因分析）
    ↓
8. WebSocket 推送实时更新
```

### 信号触发流程

```
1. 市场数据更新（WebSocket 推送）
    ↓
2. SignalDetectionService 检测信号条件
    ↓
3. 边缘触发检测（False → True 才触发）
    ↓
4. 通知 StrategyManager
    ↓
5. 更新策略 last_trigger_at
    ↓
6. 触发 AI 决策流程
```
