## 1. 快速开始

### 1.1 系统要求

- **操作系统**: macOS, Linux, Windows (WSL2)
- **Docker**: 20.10+ (包含 Docker Compose)
- **Node.js**: 18+ (前端开发)
- **Python**: 3.12+ (后端开发)
- **pnpm**: 8+ (前端包管理)

### 1.2 5 分钟快速启动

```bash
# 1. 克隆仓库
git clone https://github.com/HammerGPT/Hyper-Alpha-Arena.git
cd Hyper-Alpha-Arena

# 2. 启动应用
docker compose up -d --build

# 3. 查看日志
docker compose logs -f app

# 4. 访问应用
# 浏览器打开 http://localhost:8802
```

### 1.3 验证安装

```bash
# 检查容器状态
docker compose ps

# 应该看到:
# NAME                  STATUS         PORTS
# agentq-app            Up (healthy)   0.0.0.0:8802->8802/tcp
# agentq-postgres       Up (healthy)   0.0.0.0:5432->5432/tcp
```

## 3. 开发工作流

### 3.1 后端开发

#### 添加新的 API 端点

```python
# backend/api/my_routes.py

from fastapi import APIRouter, Depends
from sqlalchemy.orm import Session
from database.connection import get_db

router = APIRouter()

@router.get("/my-endpoint")
async def my_endpoint(db: Session = Depends(get_db)):
    """我的新端点"""
    return {"message": "Hello"}
```

注册路由：

```python
# backend/main.py

from api.my_routes import router as my_router

app.include_router(my_router)
```

#### 添加新的数据库模型

```python
# backend/database/models.py

class MyModel(Base):
    __tablename__ = "my_models"
    
    id = Column(Integer, primary_key=True)
    name = Column(String(100), nullable=False)
    
    created_at = Column(TIMESTAMP, server_default=func.current_timestamp())
```

创建迁移：

```python
# backend/database/migrations/add_my_model.py

def migrate(db):
    db.execute(text("""
        CREATE TABLE IF NOT EXISTS my_models (
            id SERIAL PRIMARY KEY,
            name VARCHAR(100) NOT NULL,
            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
        )
    """))
    db.commit()
```

#### 添加新的服务

```python
# backend/services/my_service.py

class MyService:
    def __init__(self, db: Session):
        self.db = db
    
    def do_something(self):
        # 业务逻辑
        pass
```

import pandas as pd

## 7. 常见问题

### 7.1 启动问题

#### 问题：端口 8802 已被占用

**解决方案**:

```bash
# 停止占用端口的进程
lsof -ti:8802 | xargs kill -9

# 或者修改 docker-compose.yml 中的端口映射
ports:
  - "8803:8802"  # 改为 8803
```

#### 问题：数据库连接失败

**解决方案**:

```bash
# 检查 PostgreSQL 是否运行
docker compose ps postgres

# 查看 PostgreSQL 日志
docker compose logs postgres

# 等待数据库就绪（首次启动需要初始化）
docker compose logs postgres | grep "database system is ready"
```

### 7.2 前端问题

#### 问题：页面空白

**解决方案**:

```bash
# 查看浏览器控制台错误
# F12 -> Console

# 检查 WebSocket 连接
# Network -> WS

# 清除缓存
# Ctrl+Shift+R (强制刷新)
```

#### 问题：WebSocket 连接失败

**解决方案**:

```bash
# 检查后端日志
docker compose logs app | grep WebSocket

# 检查防火墙设置
# 确保 8802 端口开放

# 测试 WebSocket 连接
wscat -c ws://localhost:8802/ws
```

### 7.3 交易问题

#### 问题：AI 决策不触发

**排查步骤**:

```bash
# 1. 检查策略配置
docker compose exec postgres psql -U alpha_user -d alpha_arena \
  -c "SELECT * FROM account_strategy_configs;"

# 2. 检查上次触发时间
# 查看 last_trigger_at 字段

# 3. 检查信号池配置
docker compose exec postgres psql -U alpha_user -d alpha_arena \
  -c "SELECT * FROM signal_pools WHERE enabled=true;"

# 4. 查看 AI 决策日志
docker compose exec postgres psql -U alpha_user -d alpha_arena \
  -c "SELECT * FROM ai_decision_logs ORDER BY created_at DESC LIMIT 10;"
```

#### 问题：订单执行失败

**排查步骤**:

```bash
# 1. 检查账户余额
docker compose exec postgres psql -U alpha_user -d alpha_arena \
  -c "SELECT current_cash, frozen_cash FROM accounts WHERE id=1;"

# 2. 检查订单状态
docker compose exec postgres psql -U alpha_user -d alpha_arena \
  -c "SELECT * FROM orders ORDER BY created_at DESC LIMIT 10;"

# 3. 查看后端错误日志
docker compose logs app | grep "order.*error"
```

## 8. 贡献指南

### 8.1 提交代码

```bash
# 1. Fork 仓库
# GitHub 上点击 Fork

# 2. 克隆 Fork
git clone https://github.com/YOUR_USERNAME/Hyper-Alpha-Arena.git

# 3. 创建分支
git checkout -b feature/my-feature

# 4. 提交代码
git add .
git commit -m "feat: add my feature"

# 5. 推送
git push origin feature/my-feature

# 6. 创建 Pull Request
# GitHub 上点击 "Compare & pull request"
```

### 8.2 提交规范

遵循 Conventional Commits:

```
feat: 新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式（不影响功能）
refactor: 重构
test: 测试相关
chore: 构建/工具相关
```

示例：

```bash
git commit -m "feat: add new momentum factor"
git commit -m "fix: correct signal trigger logic"
git commit -m "docs: update API documentation"
```

### 8.3 Code Review

提交 PR 后：

1. 等待 CI 检查通过
2. 维护者进行 Code Review
3. 根据反馈修改代码
4. 合并到主分支

### 8.4 报告问题

使用 GitHub Issues:

1. **Bug 报告**: 描述问题、复现步骤、期望行为
2. **功能建议**: 描述功能、使用场景、实现思路
3. **文档改进**: 指出不清晰的地方
